Reconciler

The following is intended soley for implementors that are interested in implementing a renderer on top of DIO's reconciler, an identical parallel can be drawn from the relationship that ReactNative™ shares with React™.


createFactory(
	type: {...}
): object

A reconciler for the DOM is shipped by default, however the createFactory API allows us to also create different host reconcilers that might target different platforms.

The creation of a factory reconciler is assumed when an object argument supplied to createFactory does not satisfy the conditions for an element factory.

This object should implement the methods it needs to render to the desired target enviroment.

For example if we were creating an reconciler for a mobile platform we might need to implement every method, on the other hand if we were doing the same for a DOM based platform, we could get away with implemnting only the methods we want to change to serve our specific need/heuristics.

Within the type comments the following {Element} does not refer to a DOM Element but instead a DIO {Element} instance. The Element instance has all the meta data for the element. The "owner" property on an instance is a readonly property reserved for referencing the host instance, additionally all {Element} instances have additional contextal meta data like "next", "prev" and "parent" to reference sibling and parent references. An additional property "cache" exists for the needs of storing any additional meta data a reconciler might want to cache.

setDocument

Called when initialiazing a mount container. Use this to setup the native view before a mount. 


setDocument: (parent: Element) => void

setText

Called when updating a text nodes value. Use this to update a native text instance value. 


setText: (element: Element, value: string) => void

setComment

Called when updating a comments nodes value. Use this to update a native comment instance value. 


setComment: (element: Element, value: string) => void

setProps

Called when assigning an elements individual properties. Use this to update or initialize properties. 


setProps: (element: Element, name: string, value: any, xmlns: string?) => void

getOwner

Called when retrieving an elements interface via findDOMNode. Use this to return a public host node or a proxy that exposes a public interface into the host enviroment. 


getOwner: (element: Element) => Node

getContext

Called when setting up a default root context. Use this to setup the root default context of a render tree. 


getContext: (element: Element) => object

getDocument

Called when retrieving a default container. Use this to reference a default container when a container is not explicity passed to render or createPortal 


getDocument: () => Node

getTarget

Called when retrieving a node interface from an event object via findDOMNode(event). 


getTarget: (event: object) => Node

getListener

Called when retrieving an events listener/callback. 


getListener: (element: Element, event: object) => function

getType

Called when retrieving an elements namespace, before an instance is created. Use this to setup an element namespace based on an elements type/props and parent namespace. 


getType: (element: Element, xmlns: string?) => string

getInitialProps

Called before commiting an elements initial props, after an instance has been created but before setProps. This is only called in the mounting phase. 


getInitialProps: (element: Element, props: object) => object

getUpdatedProps

Called before updating an elements props. Use this to define what props to update. This is only called in the update phase. 


getUpdatedProps: (element: Element, props: object) => object

getPortal

Called when retrieving a portal instance. Use this to query or find a portal instance. 


getPortal: (element: Element) => Node

getQuery

Called when hydrating a container. Use this to find a matching host instance. 


getQuery: (
	element: Element,
	parent: Element,
	previousSibling: Element,
	nextSibling: Element
) => Node

willUnmount

Called before an element and its children are unmounted. 


willUnmount: (element: Element) => boolean

isValidComponent

Called when validating a custom component class. This corresponds to createComponent. When true, createComponent is used to create the instance. 


isValidComponent: (target: object) => boolean

isValidTarget

Called when validating an instance object. 


isValidTarget: (target: object) => boolean

isValidEvent

Called when validating an event object. This is used in the context of findDOMNode(event), not to be confused with event listeners. 


isValidEvent: (event: object) => boolean

removeChild

Called when removing an instance. 


removeChild: (element: Element, parent: Element) => void

insertChild

Called when inserting an instance. 


insertChild: (element: Element, sibling: Element, parent: Element) => void

appendChild

Called when appending an instance. 


appendChild: (element: Element, parent: Element) => void

createElement

Called when creating an element instance. 


createElement: (element: Element) => Node

createComponent

Called when creating a custom component instance. 


createComponent: (element: Element) => Node

createText

Called when creating a text instance. 


createText: (element: Element) => Node

createEmpty

Called when creating an empty instance. 


createEmpty: (element: Element) => Node

createComment

Called when creating a comment instance. 


createComment: (element: Element) => Node
